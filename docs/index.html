sur is a single-binary, interactive hardening tool for fresh Linux servers. Audit, pick fixes in a TUI, apply with per-task backup + rollback, and keep every session in SQLite — so you can prove what changed and undo it when something breaks. sur, yeni Linux sunucuları için geliştirilmiş single-binary, interaktif bir hardening aracıdır. Audit çalıştır, düzeltmeleri TUI üzerinden seç, her task için backup + rollback ile uygula ve her session bilgisini SQLite içinde sakla. Böylece neyin değiştiğini kanıtlayabilir, bir şey bozulursa geri alabilirsin.

02 The four verbs Dört temel command 

$ sur check                  # audit + score
$ sur check --deep --json    # merge with lynis, machine output
$ sur harden                 # Bubble Tea TUI: pick & apply
$ sur harden --dry-run       # preview without touching anything
$ sur harden --yes           # CI / headless
$ sur rollback <session>     # reverse a previous run
$ sur history                # list past sessions

04 The task lifecycle Task lifecycle

Every task runs the same five-step state machine — and if any step fails, only that task is rolled back. The rest of the run continues. Her task aynı beş adımlı state machine üzerinden çalışır. Herhangi bir adım hata verirse sadece ilgili task rollback edilir. Run devam eder.

pre_check→ backup→ exec→ post_check→ rollback

05 A task is just YAML Task sadece YAML 

id: disable_root_ssh
name: "Disable SSH root login"
rollback_possible: true
backup_files: [/etc/ssh/sshd_config]
risk_level: low
distros: [ubuntu, debian, rocky, alma, fedora]

pre_check:
  command: "grep -Eiq '^PermitRootLogin yes' /etc/ssh/sshd_config"
  expect_exit: 0

exec:
  - command: "sed -ri 's/^#?PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config"
  - command: "systemctl restart ssh 2>/dev/null || systemctl restart sshd"

post_check:
  command: "grep -Eq '^PermitRootLogin no' /etc/ssh/sshd_config"
  expect_exit: 0

rollback:
  - command: "cp {backup_path} /etc/ssh/sshd_config"
  - command: "systemctl restart ssh 2>/dev/null || systemctl restart sshd"

{backup_path} is substituted at rollback time with the file sur backed up before applying. {backup_path}, rollback sırasında sur tarafından alınan backup dosyasıyla değiştirilir.
