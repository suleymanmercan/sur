sur is a single-binary, interactive hardening tool for fresh Linux servers. Audit, pick fixes in a TUI, apply with per-task backup + rollback, and keep every session in SQLite — so you can prove what changed and undo it when something breaks.

→ get started how it works

01Why another hardening tool

consent-first

Every change is shown, checked, and approved before it touches the system.

reversible

Each task backs up its config and registers a rollback. One command undoes a whole session.

auditable

SQLite at /var/lib/sur/sur.db: every session, every task, every backup blob.

static binary

Pure-Go SQLite driver (modernc.org/sqlite) → no CGO, no dependencies, one file deploy.

02The four verbs

$ sur check                  # built-in audit + score
$ sur check --deep --json    # merge with lynis, machine output
$ sur harden                 # Bubble Tea TUI: pick & apply
$ sur harden --dry-run       # preview without touching anything
$ sur harden --yes           # CI / headless
$ sur rollback <session>     # reverse a previous run
$ sur history                # list past sessions

04The task lifecycle

Every task runs the same five-step state machine — and if any step fails, only that task is rolled back. The rest of the run continues.

pre_check→ backup→ exec→ post_check→ rollback (on error)

05A task is just YAML

id: disable_root_ssh
name: "Disable SSH root login"
rollback_possible: true
backup_files: [/etc/ssh/sshd_config]
risk_level: low
distros: [ubuntu, debian, rocky, alma, fedora]

pre_check:
  command: "grep -Eiq '^[#[:space:]]*PermitRootLogin[[:space:]]+yes' /etc/ssh/sshd_config"
  expect_exit: 0
exec:
  - command: "sed -ri 's/^#?\\s*PermitRootLogin\\s+.*/PermitRootLogin no/' /etc/ssh/sshd_config"
  - command: "systemctl restart ssh 2>/dev/null || systemctl restart sshd"
post_check:
  command: "grep -Eq '^PermitRootLogin[[:space:]]+no' /etc/ssh/sshd_config"
  expect_exit: 0
rollback:
  - command: "cp {backup_path} /etc/ssh/sshd_config"
  - command: "systemctl restart ssh 2>/dev/null || systemctl restart sshd"

{backup_path} is substituted at rollback time with the file sur backed up before applying.
